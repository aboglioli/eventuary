# eventuary-kafka

Apache Kafka event backend for [eventuary](https://crates.io/crates/eventuary). Wraps `rdkafka` to provide a `Writer` that publishes serialized events as Kafka records (event key as record key, JSON-encoded `SerializedEvent` as payload) and a `Reader` that streams from a `StreamConsumer`, deserializes records, and emits a `Message<BatchedAcker<KafkaOffsetToken>>` whose token carries `(topic, partition, offset)`. Acks are batched and committed via `consumer.commit` with the highest contiguous offset per partition (`offset + 1`). `nack` is intentionally a no-op: offsets are simply left uncommitted, so unacked records will be redelivered after a rebalance, restart, or session expiry — there is no immediate redelivery. Reader configuration uses `kafka_topics: Vec<String>` (Kafka topic names to subscribe/assign). Event-level filtering is not performed by this reader; wrap with `FilteredReader` from `eventuary-core` if predicate-based filtering on deserialized events is needed. `StartFrom::Earliest`, `StartFrom::Latest`, and `StartFrom::Timestamp` are honored; `Specific` partition assignment bypasses the consumer group balancer. Records that fail to deserialize (missing payload, malformed JSON, invalid event) are acked and skipped as poison-record handling. Building requires `cmake` (rdkafka uses the cmake-build feature). Integration tests use [`testcontainers`](https://crates.io/crates/testcontainers) to spawn `confluentinc/cp-kafka:7.6.0` in KRaft mode; on rootless podman set `DOCKER_HOST=unix:///run/user/$(id -u)/podman/podman.sock` and `TESTCONTAINERS_RYUK_DISABLED=true` before invoking `cargo test`.
