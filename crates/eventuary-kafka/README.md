# eventuary-kafka

Apache Kafka event backend for [eventuary](https://crates.io/crates/eventuary). Wraps `rdkafka` to provide a `Writer` that publishes serialized events as Kafka records (event key as record key, JSON-encoded `SerializedEvent` as payload) and a `Reader` that streams from a `StreamConsumer`, deserializes records, and emits a `Message<BatchedAcker<KafkaOffsetToken>>` whose token carries `(topic, partition, offset)`. Acks are batched and committed via `consumer.commit` with the highest contiguous offset per partition (`offset + 1`). `nack` is intentionally a no-op: offsets are simply left uncommitted, so unacked records will be redelivered after a rebalance, restart, or session expiry — there is no immediate redelivery. Reader configuration separates `kafka_topics: Vec<String>` (Kafka topic names to subscribe/assign) from `event_topics: Option<Vec<Topic>>` (post-deserialization filter on the eventuary `Topic`); a `namespace: Option<Namespace>` filter is applied as a prefix match. `StartFrom::Earliest`, `StartFrom::Latest`, and `StartFrom::Timestamp` are honored; `Specific` partition assignment bypasses the consumer group balancer. Records whose embedded `organization` does not match the configured one are acked and skipped. Building requires `cmake` (rdkafka uses the cmake-build feature). Integration tests use [`testcontainers`](https://crates.io/crates/testcontainers) to spawn `confluentinc/cp-kafka:7.6.0` in KRaft mode; on rootless podman set `DOCKER_HOST=unix:///run/user/$(id -u)/podman/podman.sock` and `TESTCONTAINERS_RYUK_DISABLED=true` before invoking `cargo test`.
